# Development: SDLC

> Software Development Life Cycle

1. Requirements Analysis (_DEFINE_):
    - gather requirements.
    - what problems need to be solved.
    - clarify what is expected.
    - what should the finished product accomplish.
2. Planning (_DESIGN_):
    - what tech stack is required to best solve the problem.
    - plan how each technology interacts with each other.
    - consider program flow (Storyboards, flowcharts...).
3. Local Development ((_IMPLEMENT_)):
    - setup a development environment.
        - backend stack on localhost.
        - direct the output to the frontend stack in the browser.
    - use the **LAMP** or **WAMP** stack (**L**inux or **W**indows, **A**pache, **M**ySQL, and **P**HP).
4. Testing (_TEST_):
    - test cases for each moving part in the software solution.
    - choose appropriate tests that confirm both that the program works and also that it fails in predictable ways.
    - integration testing ensures that components developed separately work together as expected.
5. Remote Deployment (_DEPLOY_):
    - move the code to a _production server_.
6. Final Testing (_TEST_):
    - make sure the code behaves the same way in production as it does locally.
    - perform load testing to ensure the application can handle expected traffic.
    - ensure credentials are secure and properly validated.
